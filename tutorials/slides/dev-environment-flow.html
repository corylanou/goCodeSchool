Dev Environment Flow

A shared rhythm for getting code from idea → laptop → GitHub → production with confidence.
Use ⬅️ ➡️ arrows or the buttons below to navigate.

Session Outcomes

  • Map the full loop: clone → code → commit → PR → merge.
  • Configure VS Code + terminal so pairing feels smooth.
  • Adopt collaboration rituals that keep repos healthy.

Environment Checklist

  • Install Git, Node/Go, task runner, package manager.
  • Configure shell with aliases, direnv, prompt.
  • VS Code: settings sync, font, theme, keybindings.
Day-zero goal: every student can run npm test / go test successfully.

The Daily Loop

git pull origin main
npm install         # or go mod tidy
npm run dev         # run the app
# work in VS Code + terminal
npm test            # verify
git status          # review changes
Teach students to narrate this loop aloud while pairing.

Branch Strategy

  • Create feature branches: feat/profile-form.
  • Commit small steps with present-tense messages.
  • Rebase or merge main daily to avoid drift.
git checkout -b feat/profile-form
# code
git commit -m "Add profile form skeleton"

Pull Requests

  • Open PR early with context, screenshots, test plan.
  • Request review, respond to comments, update branch.
  • CI must pass; merge with squash for clean history.
PR template: summary, screenshots, checklist, QA steps.

VS Code Essentials

  • Command Palette (⇧⌘P / Ctrl+Shift+P) for everything.
  • Source Control panel for staging + inline diff.
  • Recommended extensions: ESLint/Go tools, Prettier, GitLens, Error Lens, Live Share.

Terminal Rituals

  • Use alias gs="git status" and alias gb="git branch".
  • Keep split panes: server logs on left, tests on right.
  • Teach tmux or VS Code integrated terminals for remote pairing.

Quality Gates

  • Run tests + linters locally before pushing.
  • Use husky/pre-commit to enforce formatting.
  • Document "Definition of Done" in CONTRIBUTING.md.

Pairing Workflow

  • Driver codes; navigator spots issues + plans ahead.
  • Swap roles every 15–20 minutes.
  • End with commit, PR draft, and short retro.
Encourage Live Share or screen share with shared terminal.

Lab: Ship a Tiny Feature

  • Clone the starter repo together.
  • Add a route/button, run tests, commit with message template.
  • Open PR, tag reviewer, walk through diff out loud.
Goal: every learner experiences the entire loop in 45 minutes.
