API Design Concepts

Ten habits to keep your Go APIs calm, confident, and production ready.

▶ Read the inspiration by Vaibhav Chichmalkar
Use ⬅️ ➡️ arrow keys or the buttons below to move through the deck.

Idempotency

  • What it is: Doing the same request twice gives the same result.
  • Why it matters: Safe retries (no double charges or duplicates).
  • Everyday example: Click “Pay” twice, still one order/charge.
Practice: Require an Idempotency-Key header on POST /orders and reuse the saved result when the key repeats.

Pagination

  • What it is: Break big lists into smaller pages or “load more”.
  • Why it matters: Keeps lists fast and smooth as data grows.
  • Everyday example: Instagram feed loads more as you scroll.
Practice: Prototype both offset and cursor styles; return a next_cursor value.

Versioning

  • What it is: Keep old and new API versions side‑by‑side.
  • Why it matters: Old apps keep working while you improve.
  • Where you’ll see it: /v1/users and /v2/users.
Practice: Route both /v1/profile and /v2/profile to different handlers in the same mux.

Rate Limiting

  • What it is: A fair speed limit for requests.
  • Why it matters: Protects servers and users during spikes.
  • Where you’ll see it: Logins, search, public endpoints.
Practice: Wrap a handler with rate.NewLimiter and log rejections to tune the ceiling.

Error Contracts

  • What it is: Predictable errors with clear codes/messages.
  • Why it matters: Faster fixes and fewer support pings.
  • Where you’ll see it: Validation, conflicts, missing fields.
Practice: Centralize error responses with a helper that sets code, message, and trace ID.

Caching

  • What it is: Reuse a saved response instead of redoing work.
  • Why it matters: Faster pages and less database load.
  • Where you’ll see it: Lists, avatars, static files.
Practice: Add an ETag header and respond 304 when the client sends If-None-Match.

Security

  • What it is: Short‑lived tokens prove who you are.
  • Why it matters: Limits risk if a token leaks.
  • Where you’ll see it: Logins, API keys, sessions.
Practice: Issue a one-minute JWT and log how your middleware handles expiry.

N+1 Queries

  • What it is: Loading related data one row at a time.
  • Why it matters: It’s slow—batch or preload instead.
  • Where you’ll see it: Users + Orders, Posts + Comments.
Practice: Measure the difference between looping queries and a single eager load.

Documentation

  • What it is: A readable contract (often OpenAPI).
  • Why it matters: Shared understanding across teams.
  • Where you’ll see it: Public APIs, onboarding, QA.
Practice: Annotate one handler with Swag tags and show the auto-generated page.

Consistency

  • What it is: Data may take a moment to sync everywhere.
  • Why it matters: Keeps systems fast and available.
  • Where you’ll see it: Feeds, counters, notifications.
Practice: Return 202 Accepted while a goroutine finishes the heavy work.

Hands-on Moves

  1. Add idempotency keys to an existing POST endpoint.
  2. Swap offset pagination for cursors on a list route.
  3. Adopt structured error responses across the service.
  4. Publish an OpenAPI snippet and review it with your frontend partner.
Pair up and share what changed in your telemetry after each improvement.
