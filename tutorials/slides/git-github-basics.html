Git & GitHub

Git tracks your code history and GitHub shares it with your crew—video game vibes are just for color.

Why We Care

  • Git records every project change so you can restore stable versions. Game parallel: feels like a playable memory of your adventure.
  • GitHub hosts that history online so teammates can sync anytime. Game parallel: everyone loads the same progress file.
  • Together they let you experiment safely and undo calmly. Game parallel: the party respawns without losing loot.
GitHub is the real website where you share repos; in game terms it’s the lobby where your party gathers before the next quest.

Git Checkpoints

  • Each commit captures the current state of your tracked files. Game parallel: like grabbing a glowing save crystal when things feel stable.
  • Every commit includes your message explaining the change. Game parallel: jotting a quick note about the mission you completed.
  • If something breaks, you can restore an earlier commit. Game parallel: reload the last save and respawn with confidence.
Pro move: Save often. Small commits beat giant “I hope this works” saves.

GitHub Basics

  • Stores a copy of your repo online so teammates can sync anytime. Game parallel: your shared server with the latest save.
  • Shows who changed what and when for clear collaboration. Game parallel: a scoreboard that awards XP to the right players.
  • Connects to VS Code so you can push and pull without the terminal. Game parallel: quality-of-life mods that streamline co-op.
Public repos showcase real work to schools, internships, and future squads—think highlight reel in game terms.

VS Code Flow

  1. Make edits in your project files.
  2. Open the Source Control tab (branch icon).
  3. Stage the files you want included in this commit (click the +). Game parallel: pack the gear you’ll carry forward.
  4. Write a short commit message: “Add hero image”.
  5. Hit the checkmark to commit, then Sync to share with the team.
VS Code runs Git commands for you. Focus on the story you’re telling with each commit.

Merge Conflicts

  • Happens when two people edit the same line, so Git pauses for a decision. Game parallel: the party has to agree before the boss battle continues.
  • VS Code shows both versions between <<<<<< and >>>>>>. Game parallel: choose which script becomes canon.
  • Chat with your teammate, edit the file to the final text, then commit again. Game parallel: strategize together, lock in the winning move.
Conflicts aren’t bad—they protect real work. Co-op like a boss fight once you’ve got the plan.

Practice Quest

  1. Create a new folder and open it in VS Code.
  2. Initialize Git in the Source Control tab.
  3. Add notes.md, write one sentence, commit, sync.
  4. Edit the note, commit again, sync.
  5. Show the commit history on GitHub—celebrate the timeline!
Run this together so everyone feels the commit loop before touching bigger projects.
