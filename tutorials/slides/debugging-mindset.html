Debugging Mindset

Bugs are feedback. Use evidence, tools, and teamwork to turn chaos into clarity.
Use ⬅️ ➡️ arrows or the buttons below to navigate.

Shift the Story

  • Treat bugs like detective work, not personal failure.
  • Write everything down—steps, clues, commands.
  • Celebrate catches early; they save future panic.
“It broke” → “We found a constraint.” Language matters.

The Loop

1. Reproduce the problem reliably
2. Observe state: debugger, logs, UI, data
3. Hypothesize: what changed? what should happen?
4. Test one idea at a time
5. Fix, verify, and add a guardrail
6. Share what you learned
Run the loop end-to-end before committing to a fix.

Reproduce Strategically

  • Capture OS, browser/runtime, branch, seed data.
  • Automate reproduction: write a failing test or script.
  • Record a 60s Loom clip so teammates can see it too.

Inspect State

  • VS Code debugger: breakpoints, call stack, watch expressions.
  • Browser DevTools: Network filters, React/Redux inspectors.
  • Backends: tail structured logs, inspect DB rows, check feature flags.

Instrument & Log

  • Prefer structured logs: logger.info('User', { id, state }).
  • Tag each log with request ID or correlation ID.
  • Strip noisy logs after the fix to protect signal later.

Debugger Boost

  • Create a launch config for the project (Node/Go/Next).
  • Use conditional breakpoints: pause only when items.length === 0.
  • Step into library code to confirm assumptions, then step out.

Form Smart Hypotheses

  • Anchor on recent changes—git diff is your friend.
  • Explain hypotheses out loud; your partner will spot gaps.
  • Log each attempt in your scratchpad to avoid loops.
“If state resets after fetch, then the reducer is returning a new object.”

Verify & Guard

  • Write a regression test or Storybook story that fails first.
  • Update documentation or README with the new edge case.
  • Share a 3-sentence recap: cause, fix, prevention.

Communication Rituals

  • Bug buddy rule: ask for help after 20 focused minutes.
  • Threaded chat updates: clues, logs, screenshots in one place.
  • Daily “duck stand-up”: everyone states what they are debugging.

Lab: Mystery Bug

  • Broken todo app—items vanish after refresh.
  • Students document repro, inspect state, patch reducer.
  • Deliver: fix commit, regression test, README note.
Timebox 30 minutes, then debrief signals that led to the fix.

Cheat Sheet

  • 401 errors → check env vars/tokens/headers.
  • UI not updating → inspect state mutation and effect deps.
  • Flaky tests → control randomness, await async, isolate side effects.
